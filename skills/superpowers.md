# Superpowers

Use this workflow when you need to think clearly before building.

## When To Use

- You are starting a feature that touches 2+ files.
- You are debugging something that is not obvious.
- You need a plan before coding.
- You are writing non-trivial logic.
- You want mockups, wireframes, or visual directions before building a UI.

## Flow

1. Brainstorm the goal, constraints, and tradeoffs.
2. Turn the outcome into a short plan.
3. For visual projects, ask for 2-3 mockup or wireframe directions before
   coding.
4. Execute one checkpoint at a time.
5. Verify with the smallest useful test.
6. Summarize what changed and what is next.

## Mockup Prompt

Use this before building a page, app, dashboard, or visual workflow:

```text
Use Superpowers brainstorming.

Here is my simple PRD:

[paste the PRD]

Before building, show me 2 or 3 simple mockup directions for version one.
If you can show visual mockups, show them. If not, describe the wireframe in
plain English: header, sections, buttons, cards, and main action.

Keep it beginner-friendly and do not write code yet.
```

## Beginner Rule

Do not skip straight into code when you are unsure what you are building. Ask
the AI to scope the work first.
